# 化工控制实训装置 PCT-100

基于 **ESP32-C3** 的化工过程控制教学实训平台，集成传感器采集、逻辑联动控制、继电器执行、WiFi 物联网通信与 PC 上位机远程监控。

---

## 硬件配置

| 模块 | 型号/规格 | 接口 |
|------|----------|------|
| 主控 | ESP32-C3-WROOM-02-N4 | — |
| 温度传感器 | DS18B20 | GPIO10 (单总线) |
| 光照传感器 | GL5516 + 10KΩ 分压（GL5516 在低端） | GPIO1 (ADC) |
| 显示屏 | 0.96" OLED SH1106 | I2C: GPIO4(SDA), GPIO5(SCL) |
| 万色灯 | WS2812 RGB LED（兼运行指示） | GPIO0 |
| 继电器1 | 标准端子 (NC/COM/NO，预留) | GPIO3 |
| 继电器2 | 标准端子 (NC/COM/NO，预留) | GPIO2 |
| 继电器3 | USB母座1 电源 (控制灯光) | GPIO6 |
| 继电器4 | USB母座2 电源 (控制风机) | GPIO7 |
| KEY1 | 自锁开关 — 对外输出总开关（仅本地操作，状态上送+显示） | GPIO20 |
| KEY2 | 自复位按键 — 短按同时开关灯光风机，长按(>1s)切换模式 | GPIO21 |
| 供电 | Type-C USB 5V / DC005 5V | LDO: ME6211A33PG→3.3V |
| 通信 | WiFi 2.4GHz + USB UART (115200) | GPIO18/19 (USB D-/D+) |

## 产品功能

| 功能模块 | 具体功能 | 说明 |
|---------|---------|------|
| 温度监测 | DS18B20 实时温度采集 | 本地显示 + MQTT 上云 |
| 光照监测 | GL5516 ADC 光照采集 | 本地显示 + MQTT 上云 |

> **光照强度换算公式**（GL5516 + 10KΩ 分压电路）：
> $$
> \text{lux} \approx \frac{(\text{反相ADC})^2}{30000}, \quad
> \text{反相ADC} = 4095 - \text{rawADC}
> $$
> - 原始 ADC 读取后先反相（暗→0，亮→4095），再平方除以 30000 得到近似 lux
> - 该公式在 ESP32 `sensor.ino` 中计算，ESP32 与上位机之间传输的值均为 lux
| OLED 显示 | 实时显示温光数据、继电器状态、WiFi/MQTT 连接状态 | I2C SH1106 |
| 万色灯指示 | WS2812 RGB 灯，指示网络状态：开机红→WiFi黄→MQTT绿 | GPIO0 |
| 按键交互 | KEY1 自锁总开关（本地，状态上送+显示）+ KEY2 自复位（短按同时开关灯光风机，长按切换模式） | 硬件电平检测 |
| 自动联动 | 温度 → 风机，光照 → 灯光 | 可配置阈值 |
| 手动控制 | 远程 MQTT 命令控制 4 路继电器 | 受 KEY1 总开关限制 |
| IoT 通信 | WiFi + MQTT，数据上报、远程控制、阈值设置 | MQTTX / PC上位机 |
| 远程控制 | PC上位机远程查看数据、控制继电器 | MQTT + JSON 协议 |

## 详细业务逻辑

### 1. 硬件按键功能定义

| 按键 | 类型 | 功能 |
|------|------|------|
| **KEY1**（GPIO20） | **自锁开关**（两态保持） | **对外输出总开关**（仅本地操作）<br>• ON（闭合→高电平）：允许继电器执行输出<br>• OFF（断开→低电平）：强制断开继电器3/4，任何控制无效<br>• 状态通过 MQTT 上送平台，并在 OLED 显示 |

| **KEY2**（GPIO21） | **自复位按键**（按下导通） | **短按**（<1s）：同时切换继电器3（灯光）和继电器4（风机）ON/OFF<br>**长按**（≥1s）：切换自动/手动模式 |
> ⚠ KEY1 = OFF 时，无论自动联动、KEY2、MQTT 远程命令，继电器3/4均强制关闭。
> 这是硬件层面的安全总闸。

### 2. 系统工作模式

| 模式 | 切换方式 | 说明 |
|------|---------|------|
| **自动模式** | KEY2 长按（≥1s）切换至此 | 传感器自动采集 → 阈值判断 → 自动联动继电器3/4 → 更新 OLED → MQTT 上报 |
| **手动模式** | KEY2 长按（≥1s）切换至此 | 自动联动暂停，通过 KEY2 短按或 MQTT 远程命令控制继电器3/4 |

### 3. 联动控制规则（自动模式 + KEY1=ON 时执行）

#### ① 温度 → 继电器4（GPIO7，USB母座2）— 控制风机

| 条件 | 继电器4 动作 | 含义 |
|------|------------|------|
| 温度 > 上限阈值 | **ON** | 启动风机（散热冷却） |
| 温度回落 ≤ 上限 − 回差 | **OFF** | 停止风机 |
| 回差 | 2°C | 防止频繁通断 |

#### ② 光照 → 继电器3（GPIO6，USB母座1）— 控制灯光

| 条件 | 继电器3 动作 | 含义 |
|------|------------|------|
| 光照强度 < 阈值 | **ON** | 开启灯光（补光） |
| 光照强度 ≥ 阈值 | **OFF** | 光照充足，关闭灯光 |

> 继电器1（GPIO3）和继电器2（GPIO2）为标准端子输出，预留用于后续扩展，不参与当前业务联动逻辑。

### 4. 万色灯网络状态指示

| 设备状态 | 万色灯颜色 | 说明 |
|---------|-----------|------|
| 开机 / 初始化 | **红色** | 系统启动中 |
| WiFi 已连接 | **黄色** | 网络就绪，MQTT 连接中 |
| MQTT 已连接 | **绿色** | 物联网通信正常 |

```

┌──────────────────────┐

│  传感器采集 / OLED 显示 │
```

### 6. KEY2 操作逻辑

```
KEY2 动作
    │
    ▼
┌──────────────┐
│  检测按下时长  │
└──────┬───────┘
       │
  短按  │              长按
  (<1s) │              (≥1s)
       │                  │
┌──────▼──────┐      ┌────▼──────┐
│ 是否手动模式? │      │ 切换工作模式 │
└──────┬──────┘      │ 自动↔手动   │
       │              │ OLED更新    │
   是  │   否(自动)    └─────────────┘
┌──────▼──┐  ┌──▼─────────┐
│ 检测KEY1│  │ KEY2 无效  │
└──────┬──┘  │(自动模式)   │
       │     └────────────┘
   ON  │   OFF
┌──────▼──┐  ┌──▼─────────┐
│ 同时切换  │  │ KEY2 无效  │
│ 继电器3/4 │  │(总闸断开中)│
│ 灯光+风机 │  └────────────┘
└─────────┘
```

- **短按（<1s）**：仅在手动模式 + KEY1=ON 时，同时翻转继电器3和继电器4状态（都开→都关 / 都关→都开）
- **长按（≥1s）**：切换自动/手动模式，不受 KEY1 和当前模式影响

### 8. MQTT 通信协议

> **MQTT TCP（设备/PC端）：**`47.98.170.180:8081`  
> **MQTT WebSocket（PC上位机）：**`47.98.170.180:8083`  
> **MQTT 协议版本：**`3.1.1`（`protocolVersion: 4`）  
> **用户名：**`dzdx_emqx`  
> **密码：**`Jp4!sQ7$`  
> **默认 WiFi：**`admin / 123456`  
> `device_id` 为设备表面丝印序列号，每台设备唯一标识。

#### 数据上报 Topic（设备 → 服务器）

```
Topic: chemctrl/{device_id}/status
Payload (JSON):
{
  "temperature": 25.3,
  "light": 680,
  "mode": "auto",
  "key1_lock": true,
  "relay3": false,
  "relay4": true,
  "temp_threshold": 40.0,
  "light_threshold": 1000
}
```
#### 远程控制 Topic（服务器 → 设备）

```
Topic: chemctrl/{device_id}/command
Payload (JSON):
{
  "cmd": "set_relay",
  "relay": 3,
  "value": true
}
```

| 命令 | 参数 | 说明 |
|------|------|------|
| `set_relay` | relay: 3-4, value: true/false | 控制继电器（受 KEY1 约束；relay3 灯光，relay4 风机） |
| `set_mode` | mode: "auto"/"manual" | 切换工作模式 |
| `get_status` | — | 查询设备状态，设备收到后立即上报一次 `chemctrl/{id}/status` |
| `set_threshold` | temp: 数值, light: 数值 | 设置温度高值阈值和光照阈值 |
| `reboot` | — | 远程重启设备 |

### 10. PC 桌面控制端 `1_Software/pc/`

Electron 桌面应用，主进程通过 `mqtt` npm 包直连 MQTT TCP（`mqtt://47.98.170.180:8081`）。

**功能：**
- 自动连接 MQTT 服务器，设备编号固定为 `001`
- 实时显示温度 / 光照传感器数据
- 显示模式、总闸、继电器状态
- 切换自动 / 手动模式
- 手动模式下开启 / 关闭灯光（继电器3）和风扇（继电器4）
- 下发温度阈值和光照阈值

**文件结构：**
- `main.js` — Electron 主进程（MQTT TCP 连接、窗口管理）
- `preload.js` — IPC 桥接层（contextBridge 隔离）
- `index.html` / `css/style.css` / `js/app.js` — 渲染进程 UI
- `package.json` — 依赖配置

**使用方法：**
```bash
cd 1_Software/pc
npm install
npm start        # 开发运行
npm run dist     # 打包为 exe
```
